How to use

Quick guide
  1. Enter the property address.
  2. Set the date range and search radius.
  3. (Optional) Set a claim date — events within ± window get highlighted.
  4. Click Search. Tool geocodes the address, identifies counties, and pulls live LSR data.
  5. For finalized SED data, download from NCEI for the listed counties and drop the CSVs onto the SED upload box. The tool merges them with LSR using a cutoff (SED is authoritative through its latest report; LSR fills the gap).
  6. Use the tabs to switch between Dashboard, Map, Table, and Raw Data.
About the data

NWS LSR (Local Storm Reports) — real-time spotter reports submitted to the issuing NWS Weather Forecast Office. Pulled live via Iowa State's IEM API.

NOAA SED (Storm Events Database) — finalized, curated records published by NCEI with a 2–4 month lag. Includes events SED considers significant. Downloaded as yearly CSV by county.

Cutoff merge: SED entries take priority through SED's latest BEGIN_DATE; LSR entries fill the gap after that. This avoids double-counting while keeping the most recent weeks/months covered.

About this tool

This is a forensic-research tool for property insurance claims. It parses two storm-event sources and helps you understand which weather events occurred close to a property and when.

Data sources

NWS Local Storm Reports (LSR) — real-time, spotter-submitted reports collected by NWS Weather Forecast Offices. The tool fetches them live from Iowa Environmental Mesonet's archive (going back to ~2003). LSRs may be preliminary and contain estimation errors.

NOAA Storm Events Database (SED) — finalized, curated event records published by NCEI with a 2–4 month lag. The tool auto-fetches per-state SED CSVs from SED_BASE_URL on every search. SED_BASE_URL auto-detects from the page's URL when deployed; the data files are produced by fetch_sed.py via the bundled GitHub Action that runs monthly. SED + LSR merge via a cutoff: SED is authoritative through its latest BEGIN_DATE; LSR fills the gap after that.

County coverage

SED data is organized by county. When a property is near a county line, events in adjacent counties may still fall within the search radius. To handle this, the tool detects the property's county and the counties touching the search circle, then prompts you to download SED for all of them.

Coordinate system / projections

Distances are computed using the Haversine formula (great-circle, mean Earth radius 3,958.8 mi). For radii under ~50 mi this is accurate within tenths of a mile.

Per-day estimates (Estimates tab)

The Estimates tab emits one row per day per event type (Hail / Wind) — but only for days where at least one report of that type fell within 5 mi of the property. Each estimate is computed as follows:

  1. Inverse Distance Weighting (IDW) with exponent p=2: each report's contribution is weighted by 1/distance². The 0.5 mi epsilon prevents an at-property report from dominating completely.
  2. Adaptive radius: starts at 5 mi; if fewer than three reports are inside, expands to 10 → 15 → 20 mi. The radius actually used is shown so you can judge how local the estimate really is.
  3. Storm-track inference: with ≥3 reports the tool fits a linear regression of (lat, lon) ~ time and reports the inferred storm motion direction. The perpendicular distance from the property to that track ("path proximity") is reported. If the property is on or very close to the inferred track, that strongly suggests the storm passed over the location.
  4. Confidence (Low / Medium / High) blends closest-report distance, report count, path proximity, and whether the adaptive radius had to expand.

Limitations: storm direction is *inferred* from the report sequence — actual wind-direction observations (HRRR/RAP forecasts, METAR) are not yet pulled. UTC day grouping can split a single storm across two rows when it crosses midnight UTC.

Output formats

  • Dashboard: KPIs and charts across the full result set.
  • Map: Property + radius rings (5/10/15 mi + your custom) + event markers, color-coded by type.
  • Table: Sortable, filterable list with date, type, magnitude, distance, severity, source.
  • Estimates: Per-day IDW estimates at the property for hail size and wind speed (see methodology above).
  • Raw Data: Original LSR JSON, original SED CSV (passthrough), and a merged Combined CSV.
  • Excel export: Multi-sheet .xlsx workbook (Report, Combined, LSR, SED) for archival.
  • Print: Browser print → PDF for one-page claim reports.

Limitations

  • U.S. addresses only (Census Geocoder).
  • SED isn't queried automatically — you upload the CSVs. A backend would be required for fully-automated SED.
  • LSR magnitude isn't always populated (e.g., damage-only reports).
  • Search history is per-browser via localStorage — not synced across devices.
